# Definition of Done

## Уровень 1 - фича технически существует

- код собран
- happy-path работает

Этого недостаточно.

## Уровень 2 - фича пригодна к внутреннему использованию

- есть loading/empty/error states
- есть permission handling
- есть базовые тесты
- есть audit/logging где нужно

## Уровень 3 - фича готова к пилотным клиентам

- есть метрики использования
- проверены failure paths
- есть rollback
- есть документация для ops/support
- нет критических слепых зон по данным и доступам

## Уровень 4 - фича достойна серьёзной CRM

- влияет на ежедневную операционную работу
- формирует единый источник истины
- поддерживает командный сценарий, а не только одиночный demo-flow
- не разваливается от плохих данных, задержек сети и многопользовательской работы
- наблюдаема через логи, метрики и алерты
- не стыдно показать пилотному платящему клиенту без ручного шаманства после релиза

## Общая формула Done

Фича считается готовой только если одновременно выполнены:

1. Product fit
2. UX completeness
3. Domain correctness
4. Security/permissions correctness
5. Telemetry and auditability
6. Test coverage on risky paths
7. Release and rollback readiness
